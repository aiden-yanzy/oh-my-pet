# Phase 1.5: 经典拓麻歌子交互重构

> 基于拓麻歌子（1996-2008）交互模型重构 MVP 体验。
> 核心理念：扁平操作、即时反馈、后果机制替代冷却计时器。

## 问题回顾

1. **操作层级太深**：喂食/玩耍需要展开 popover 选类型，不符 "3键2秒" 经典模型
2. **冷却计时器无意义**：30s 倒计时不像游戏机制，像 bug
3. **任务不自动计数**：必须通过 InteractionBar 操作才触发，store 间无自动 watch
4. **探索交互差**：点了没反馈，完成后也不自动展示结果
5. **宠物无互动感**：只有一个圆 + 呼吸动画，不能点击交互，无状态切换

## 改动方案

### 1. InteractionBar → 扁平化操作栏

**当前问题：** 4 个按钮每个展开 popover → 选择 → 确认，4步操作

**改为：**
```
┌─────────────────────────────┐
│ [🍖喂食] [🎮玩耍] [🚽清洁] [🔍探索] │ ← 一行直接操作按钮
│ [😴 睡觉]  ← 切换开关 (on/off)     │
└─────────────────────────────┘
```

- **喂食**：直接喂！不弹菜单（按食物列表循环或喂默认食物）
- **玩耍**：直接玩！随机选一个游戏类型
- **清洁**：新增按钮。当宠物脏了（便便/污渍）时高亮提醒，点击清理
- **探索**：直接触发当前等级解锁的最新场景
- **睡觉**：开关按钮。开灯/关灯，代替 `rest` 的时间选择器

**详细操作流程：**
- 喂食：点击 🍖 → 宠物吃动画 (2s) → 饱腹度增加 → 过一会儿产生便便
- 玩耍：点击 🎮 → 随机选一个游戏 → 立即结算奖励 → 显示 +EXP / +属性
- 探索：点击 🔍 → 宠物离开（屏幕显示"探索中..."）→ 5-15分钟→ 完成后弹出日志
- 清洁：点击 🚽 → 便便消失 → 清洁度恢复
- 睡觉：点击 😴 → 切换睡眠状态 → 宠物闭眼 → 缓慢恢复

### 2. 移除冷却计时器，替换为后果机制

**移除：** 30s 喂食冷却、30s 游戏冷却、30min 探索冷却

**改为：**
- 喂食无限制，但：
  - 每喂食 3 次 → 产生便便（屏幕上显示 💩）
  - 便便超过 15 分钟未清理 → 宠物生病（健康-5/分钟）
  - 生病超过 30 分钟未喂药 → 宠物状态极差（动画变蔫）
- 游戏无限制，但：
  - 连续玩 5 次 → 宠物疲劳（快乐-5/次）
  - 需要睡觉恢复
- 探索无冷却，但：
  - 探索期间无法喂食/玩耍/清洁
  - 探索期间宠物不在家

**核心变化：** 从"计时器不让做"变成"做了有后果"——这和经典拓麻歌子一样，玩家自己决定节奏。

### 3. Task 自动计数

**问题：** `updateProgress` 只在 InteractionBar 里调用，如果 petStore 直接调用就漏了

**修复：** 在任务页面组件里添加 `watch`：
```typescript
// DailyTaskList.vue or taskStore
watch(() => petStore.pet?.stats.experience, () => {
  // 检测到经验变化 = 发生了喂食/玩耍/探索
  taskStore.refreshFromPetActions();
});
```
或更干净：petStore 的 feed/play/rest/explore 内部直接调用 taskStore.updateProgress

### 4. 宠物动画状态机

**当前：** 一个圆 + bob 动画

**改为多状态：**
```
idle → 呼吸动画（当前 bob）
eating → 上下快速动 + 咀嚼文字/表情（2秒后回到 idle）
happy → 跳跃 + 闪光（状态变化时触发）
sleeping → 闭眼 + 身体静止 + Zzz 文字（睡觉模式）
sick → 蔫垂 + 颜色变暗（健康 < 30）
poop → 宠物旁边显示 💩 图标（需要清洁）
click → 点击宠物弹出反应表情（😊/😡/😴 取决于状态）
```

使用 Pixi.js Graphics 绘制不同状态形状，不需要精灵图。

### 5. 状态始终可见

**当前：** Happy/Health 在 canvas 下方，需要看的时候才看到

**改为：** 精简到宠物底部图标（拓麻歌子风格）：
```
┌──────────────────┐
│    🐤 (宠物)      │
│  😊❤️💩  🍖🎮🔍  │ ← 状态区 + 操作区合并
└──────────────────┘
```

- 😊 = 快乐值（满→绿，中→黄，低→红）
- ❤️ = 健康值
- 💩 = 需要清洁（闪烁提醒）
- 等级和 EXP 条保留，但缩小

### 6. 探索流程改进

**当前：** 点击探索 → setTimeout 等 → 手动打开抽屉看日志

**改为：**
1. 点击 🔍 → 宠物离开画面 → 显示"探险中..." + 剩余时间
2. 探索完成 → 自动弹出动画日志（逐条展示事件，类似开盲盒）
3. 日志展示完后自动结算奖励
4. 探索期间宠物不可操作（但可以切到任务/档案页）

## 实现范围

不涉及：
- 后端/数据库
- 新 API
- 新页面（只在现有组件内改造）
- 进化系统改动

涉及：
- `InteractionBar.vue` → 重构（删除 popover，扁平化按钮）
- `PetCanvas.vue` → 状态图标整合到 canvas 区域
- `usePetRenderer.ts` → 动画状态机
- `petStore.ts` → 添加后果机制（poop/sick），移除冷却
- `taskStore.ts` → 自动 watch 计数
- `HomePage.vue` → 布局微调

## 优先级顺序

**P0（先做，核心体验）：**
1. 扁平化操作栏（去 popover）
2. 移除冷却计时器
3. 宠物点击互动 + 动画状态机（idle/eating/sleeping）

**P1（重要）：**
4. 后果机制（便便/生病）
5. 状态始终可见
6. 任务自动计数

**P2（锦上添花）：**
7. 探索流程改进
8. 清洁按钮加入
