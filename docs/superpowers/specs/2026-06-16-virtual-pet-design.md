# 电子宠物Web应用 - 设计文档

**创建时间：** 2026-06-16  
**最后更新：** 2026-06-16  
**版本：** 1.1 (MVP设计 - 基于调研优化)

## 1. 项目概述

### 1.1 核心定位
一款轻松佛系、支持持续成长和社交分享的像素风电子宠物PWA应用。致敬经典拓麻歌子，但去除"责任压力"，增加现代化的无限进化和社交元素。

### 1.2 目标用户
- 怀旧玩家（体验过90年代电子宠物）
- 轻度休闲玩家（碎片时间养成）
- 儿童用户（通过手机或儿童手表访问）

### 1.3 核心价值主张
- **轻松无压力**：离线不会导致宠物状态恶化或死亡
- **持续成长感**：无限进化机制，没有"最终形态"
- **惊喜探索**：回来查看宠物离线期间的探险日志
- **社交分享**：展示独特的进化路线和成就

## 2. 开发策略

### 2.1 渐进式开发
采用两阶段开发策略，先验证核心玩法，再投入完整基础设施。

**第一阶段（MVP，2-3周）：**
- 纯前端PWA + localStorage
- 核心养成循环验证
- 2-3条进化路线示例
- 基础任务和探索系统

**第二阶段（玩法验证后）：**
- 接入后端服务（Supabase）
- 用户系统和云存储
- 完整社交功能
- 扩展进化树和事件池

### 2.2 技术栈（第一阶段）

**项目脚手架：** alexanderop/vue-template (MIT)
- 完整的Vue3 + TypeScript + PWA配置
- 已包含Pinia、Vue Router、Tailwind v4、VueUse
- IndexedDB示例可直接改造为宠物数据层
- 完善的测试配置（Vitest + Playwright）

**前端框架：** Vue 3 + Vite
- 选择理由：渐进式框架，模板语法直观，适合快速开发

**状态管理：** Pinia
- 管理宠物状态、任务、事件日志

**数据持久化：** IndexedDB (优先) / localStorage (降级)
- IndexedDB：容量更大（>50MB），结构化查询，异步操作
- localStorage：简单键值对，同步操作，作为IndexedDB不可用时的降级方案
- 存储宠物数据、用户进度、AI缓存
- 迁移策略：设计数据结构时预留云端同步字段

**游戏渲染：** Pixi.js 8 (需集成)
- 高性能2D Canvas渲染引擎
- 精灵动画系统
- 粒子效果支持

**PWA配置：**
- Service Worker：离线访问支持
- Web App Manifest：可添加到主屏幕
- 图标和启动画面

**AI增强（可选但推荐）：**
- DeepSeek V3 API ($0.28/$0.42 per 1M tokens)
- 用途：探索事件生成、离线日志摘要、宠物性格描述
- 缓存策略：IndexedDB缓存常见响应，减少90%+重复请求
- 降级方案：API不可用时使用预设文本模板

**美术资源：**
- 像素风（8-bit/16-bit风格）
- AI生成基础素材 + 手工精修
- 简单帧动画（3-5帧循环）

## 3. 核心系统设计

### 3.1 宠物系统

#### 3.1.1 起始形态
- 单一起始形态：一个简单的像素风小生物
- 中性设计，不预设性别或明确物种
- 初始等级：1级
- 初始属性值均为基础值
- **DNA系统**：创建时生成隐藏基因（影响进化倾向和属性上限），让每只宠物独一无二

#### 3.1.2 属性系统

**基础属性（影响宠物状态）：**
- **快乐度**（Happiness）：0-100，影响互动响应
- **健康度**（Health）：0-100，影响活动能力
- **经验值**（Experience）：累积升级
- **等级**（Level）：1级起，无上限

**成长属性（影响进化方向）：**
- **力量**（Strength）：影响力量型进化
- **智力**（Intelligence）：影响智力型进化
- **敏捷**（Agility）：影响敏捷型进化
- **魅力**（Charm）：影响社交型进化（第二阶段）

**隐藏属性（影响个性和进化潜力）：**
- **DNA基因值**（创建时随机生成，不可见）
  - 力量基因（0.8-1.2）：影响力量属性成长速度和上限
  - 智力基因（0.8-1.2）：影响智力属性成长速度和上限
  - 敏捷基因（0.8-1.2）：影响敏捷属性成长速度和上限
  - 性格倾向（5种：活泼/文静/贪吃/高冷/好奇）：影响动画表现和AI对话风格

**属性变化规则：**
- 基础属性：通过互动维持，离线期间缓慢自然恢复
- 成长属性：通过特定互动和探索事件提升，不会下降
- 成长属性受DNA基因影响：实际获得 = 基础值 × 对应基因系数
- 经验值：所有互动和探索都会获得

#### 3.1.3 无限进化机制

**核心理念：** 持续成长，没有"最终形态"，但有阶段性的形态变化和能力解锁。

**进化维度：**

1. **等级成长**
   - 每5级一个小里程碑（解锁新互动选项）
   - 每10级触发外观微调（颜色变化、细节增加）
   - 每20级触发形态演变（显著外观变化）

2. **属性突破**
   - 某成长属性达到里程碑值（50/100/200/500...）触发对应分支强化
   - 例如：力量达到100 → 体型变大，解锁"强壮"被动技能
   - 不同属性可以同时突破，形成混合特性

3. **技能解锁**
   - 通过等级、属性、特殊事件解锁技能
   - 技能类型：
     - **被动技能**：永久效果（如"快速恢复"：离线恢复速度+50%）
     - **互动技能**：新的互动方式（如"深度睡眠"：休息效果翻倍）
     - **探索技能**：影响探索结果（如"寻宝直觉"：稀有事件概率+20%）

4. **形态演变**
   - 基于主导属性决定演变方向
   - 示例路线（第一阶段实现2-3条）：
     - **力量型**：体型增大，肌肉线条，温暖色调
     - **智力型**：头部比例增加，眼睛变大，冷色调
     - **敏捷型**：身形修长，四肢灵活，鲜艳色彩
   - 形态演变不锁死方向，后续仍可培养其他属性

**进化触发条件（复合机制，参考Pokémon/Digimon设计）：**

| 进化类型 | 触发条件示例 | 借鉴来源 |
|---------|-------------|---------|
| **等级进化** | 等级达到10/20/30... | Pokémon基础模式 |
| **属性进化** | 某属性≥100且等级≥15 | Pokémon Tyrogue模式 |
| **道具进化** | 使用特殊进化石/道具 | Pokémon进化石 |
| **事件进化** | 完成特定探索事件链 | Digimon特殊条件 |
| **复合进化** | 等级≥20 且 力量≥150 且 拥有"强壮"技能 | 多条件组合 |
| **时间进化** | 白天/夜晚进化（基于系统时间） | Pokémon Eevee→Espeon/Umbreon |
| **稀有变异** | 低概率触发（1-5%），外观特殊配色 | Pokémon Shiny / Boing Boing Animals |

**第一阶段进化示例：**
- 实现力量和智力两条路线
- 10级：基础形态进化（仅需等级）
- 20级：分支形态进化（等级 + 主导属性≥100）
- 3-5个可解锁技能
- UI展示完整进化路线图（未实现部分灰色锁定）
- 预留稀有变异槽位（第二阶段实现）

### 3.2 互动系统

#### 3.2.1 基础互动

**喂食（Feed）：**
- 消耗：食物道具（通过任务或探索获得）
- **食物类型差异化**（增强策略深度）：
  - **肉类**（牛排、鸡肉）：健康+20，力量+3，经验+5
  - **蔬果**（苹果、胡萝卜）：健康+15，智力+3，经验+5
  - **零食**（饼干、糖果）：快乐+10，健康+5，敏捷+2，经验+3
  - **特殊食物**（稀有）：健康+30，随机属性+5，经验+10
- 冷却：无，但过度喂食无额外收益
- 动画：宠物吃东西，满意表情（根据性格倾向有不同表现）

**玩耍（Play）：**
- 类型：
  - **力量游戏**（拔河、举重）：快乐+15，力量+2，经验+5
  - **智力游戏**（解谜、记忆）：快乐+15，智力+2，经验+5
  - **敏捷游戏**（追逐、跳跃）：快乐+15，敏捷+2，经验+5
- 冷却：每种游戏10分钟冷却
**玩耍（Play）：**
- 类型：
  - **力量游戏**（拔河、举重）：快乐+15，力量+2，经验+5
  - **智力游戏**（解谜、记忆）：快乐+15，智力+2，经验+5
  - **敏捷游戏**（追逐、跳跃）：快乐+15，敏捷+2，经验+5
- 冷却：每种游戏10分钟冷却
- 动画：对应游戏的简单像素动画
- **属性增长受DNA基因影响**：实际获得 = 基础值 × 基因系数

**休息（Rest）：**
- 触发：玩家点击休息按钮
- 效果：加速时间流逝，快速恢复基础属性
- 持续：可选1小时/3小时/8小时
- 期间：宠物睡眠动画，玩家可以离开

**探索（Explore）：**
- 触发：玩家点击探索，或离线自动触发
- 持续：5-15分钟（随机）
- 期间：宠物离开屏幕探险
- 结束：生成探索日志和奖励
- 冷却：30分钟

#### 3.2.2 探索与随机事件

**探索场景系统（参考TamaGROK设计，第一阶段实现2-3个场景）：**

| 场景 | 解锁条件 | 特点 | 事件倾向 |
|------|---------|------|---------|
| **草原** | 默认 | 基础场景，平衡 | 发现食物、遇到小动物 |
| **森林** | 等级≥10 | 智力导向 | 解谜、学习技能、发现草药 |
| **山地** | 等级≥15 | 力量导向 | 攀岩挑战、发现矿石、体能训练 |
| **河流** | 等级≥20（第二阶段） | 敏捷导向 | 游泳、捕鱼、水上竞速 |
| **神秘洞穴** | 特殊条件（第二阶段） | 稀有奖励高 | 宝箱、稀有进化道具 |

**事件生成机制（混合模式：预设 + AI增强）：**

**模式一：预设事件库（第一阶段主要方式，成本$0）**
- 每个场景10-15个手写事件
- 权重随机选择
- 快速、稳定、无成本

**模式二：AI动态生成事件（可选，增强体验）**
- **触发条件**：预设事件已被玩家体验3次以上，或特殊里程碑（等级10/20/30）
- **生成策略**：
  ```
  输入：场景类型 + 宠物等级 + 主导属性 + 性格倾向 + 最近3次事件摘要
  输出：JSON结构化事件（标题、描述、奖励、选项）
  模型：DeepSeek V3 API
  ```
- **缓存策略**：
  - 生成的事件存入IndexedDB，标记为"已缓存事件"
  - 相同条件下优先使用缓存（命中率预期70%+）
  - 每个场景缓存池维持20-30个事件变体
- **Fallback**：API失败或超时(>3秒)时回退到预设事件
- **成本控制**：每次生成~500 tokens，缓存后90%+请求无需API调用
  - 预期月成本（1000 DAU）：$10-15

**事件权重分布（预设事件池）：**

**常见事件（70%概率）：**
- 发现食物（获得对应场景的特色食物）
- 发现玩具（获得玩具道具）
- 遇到小动物（快乐+10，魅力+1，性格影响对话）
- 场景特定训练（对应属性+2-5）
- 天气事件（晒太阳/淋雨，健康/快乐±5）

**稀有事件（25%概率）：**
- 发现宝箱（大量经验或稀有道具）
- 学习新技能（解锁条件满足时触发）
- 遇到导师NPC（主导属性+10）
- 特殊场景发现（记录到图鉴，第二阶段展示用）
- 分支选择事件（玩家选择影响属性成长方向）

**关键事件（5%概率）：**
- 属性突破选择（玩家二选一，影响进化方向）
- 形态进化触发（满足条件时，必须在线）
- 稀有技能解锁机会
- 进化道具发现（进化石、特殊物品）

**离线探索机制：**
- 每2小时自动触发一次探索
- 离线期间积累探索结果（上限7天，最多84次）
- 回来时生成"离线日志摘要"：
  - **预设模式**：简单文本模板填充
  - **AI增强模式**（可选）：DeepSeek根据离线期间的事件序列生成1-3句叙事摘要
    - 示例：*"你离开的这三天，小宝在草原上遇到了新朋友，还学会了在森林里寻找蘑菇。它现在看起来更聪明了！"*
    - 成本极低：每次~200 tokens，批量生成，1000 DAU月成本~$3-5
- 批量展示日志（最多显示最近5条详细事件）
- 所有奖励自动结算，关键事件暂停等待玩家确认

### 3.3 任务系统

#### 3.3.1 日常任务

每日凌晨0点刷新，3-5个简单任务：

**任务示例：**
- 喂食宠物3次（奖励：食物×2，经验+20）
- 完成任意游戏5次（奖励：玩具×1，经验+30）
- 完成1次探索（奖励：随机道具，经验+25）
- 宠物达到满快乐或满健康（奖励：经验+15）
- 累计在线时长10分钟（奖励：食物×1）

**任务奖励：**
- 经验值
- 道具（食物、玩具）
- 第二阶段：货币、限定装饰

#### 3.3.2 里程碑成就（第二阶段）

长期目标记录：
- 养成天数（7天/30天/100天...）
- 等级达成（10级/50级/100级...）
- 技能解锁数量
- 探索次数
- 社交互动次数

### 3.4 离线机制

#### 3.4.1 核心原则
**轻松佛系**：离线不惩罚玩家，但关键时刻需要玩家在场。

#### 3.4.2 离线行为

**自动维护：**
- 基础属性（快乐、健康）缓慢自然恢复到70%
- 不会因离线而降至危险值或死亡
- 记录离线时长

**自动探索：**
- 每2小时触发一次探索
- 积累探索结果和奖励
- 普通事件自动结算
- 关键事件暂停等待玩家

**暂停机制：**
- 以下事件离线期间不触发，等待玩家在线：
  - 形态进化
  - 技能解锁选择
  - 关键剧情事件
  - 属性突破确认

**回归体验：**
- 显示离线时长摘要
- 播放"宠物日志"（最近5条事件）
- 展示所有获得的奖励
- 提示暂停的关键事件

## 6. 游戏平衡与数值设计

### 6.1 升级曲线

**经验需求：**
- Level 1→2: 100 XP
- Level 2→3: 150 XP
- Level 3→4: 225 XP
- 公式：`nextLevelXP = baseLevelXP * (1.5 ^ (level-1))`
- 平滑增长，避免后期过度肝

**经验获取：**
- 喂食：5 XP
- 玩耍：5 XP
- 完成任务：15-30 XP
- 探索普通事件：10 XP
- 探索稀有事件：25-50 XP

### 6.2 属性成长速度

**成长属性增长：**
- 针对性互动：+2/次
- 随机探索事件：+1~5
- 属性突破里程碑：50/100/200/500/1000...
- 设计目标：玩家每天正常游玩，一周可达到第一个属性突破

### 6.3 时间平衡

**单次游戏时长：**
- 快速互动（喂食、玩耍）：10-30秒
- 探索：5-15分钟（可离线）
- 每日任务：累计5-10分钟可完成

**每日预期投入：**
- 轻度玩家：3-5分钟（完成基础互动）
- 中度玩家：10-15分钟（完成所有日常任务）
- 重度玩家：随时查看探索日志，频繁互动

**离线友好：**
- 1天不上线：自动恢复基础属性，积累~12次探索
- 1周不上线：仍可获得离线期间所有奖励，不会失去进度

## 10. AI增强系统（可选但推荐）

### 10.1 设计哲学

**核心原则：** AI是"增强"而非"替代"。预设内容保证基础体验，AI锦上添花提供个性化和新鲜感。

**渐进式引入策略：**
- Phase 1（MVP）：纯预设内容，验证核心玩法，成本$0
- Phase 2（AI试点）：离线日志摘要 + 部分探索事件AI生成，月成本$15-20
- Phase 3（AI深化）：宠物性格对话系统，月成本$30-50

### 10.2 技术选型

**模型选择：DeepSeek V3**
- **定价**：$0.28 input / $0.42 output per 1M tokens（2026年价格）
- **选择理由**：
  - 性价比高于GPT-4o mini ($0.60 output)
  - 质量优于Gemini 2.0 Flash（实测中文理解更好）
  - 响应速度快（适合实时场景）
  - 支持结构化输出（JSON mode）

**备选方案：**
- **Gemini 2.0 Flash** ($0.10/$0.40)：预算极度紧张时的替代
- **本地模型**（Qwen 2.5 3B）：DAU > 5000后考虑部署，边际成本趋近$0

### 10.3 功能实现详解

#### 10.3.1 离线日志AI摘要 ⭐⭐⭐⭐⭐

**优先级：最高**（成本最低、价值高、实现简单）

**触发时机：** 玩家离线≥4小时后首次回归

**输入数据：**
```javascript
{
  offlineHours: 8,
  explorationCount: 4,
  events: [
    { type: "found_food", scene: "grassland", item: "apple" },
    { type: "met_animal", scene: "forest", animal: "squirrel" },
    { type: "learned_skill", skill: "quick_recovery" },
    { type: "attribute_increase", attr: "intelligence", value: 5 }
  ],
  petName: "小宝",
  personality: "curious",
  levelBefore: 8,
  levelAfter: 9
}
```

**Prompt模板：**
```
你是一个电子宠物游戏的叙事引擎。根据以下数据，生成1-3句温馨的离线日志摘要。

宠物信息：
- 名字：{petName}
- 性格：{personality}（好奇/活泼/文静/贪吃/高冷）
- 离线时长：{offlineHours}小时

离线期间发生的事件：
{events_summary}

要求：
1. 用第三人称叙述（"小宝在..."）
2. 语气温暖、有画面感
3. 突出重要事件（技能解锁、等级提升）
4. 控制在50字以内
5. 不要使用emoji

输出格式：纯文本（无需JSON）
```

**缓存策略：**
- 相同事件组合缓存结果（key = 事件类型序列的hash）
- IndexedDB存储，TTL = 7天
- 预期缓存命中率：60-70%

**成本估算：**
- 输入：~300 tokens，输出：~80 tokens
- 未缓存时每次成本：$0.000084 + $0.000034 = $0.000118
- 1000 DAU，每天1次，缓存命中70% → 月成本 = 1000 × 30 × 0.3 × $0.000118 ≈ **$1.06/月**

## 11. 风险与挑战

### 10.1 设计风险

**风险：** 无限进化可能导致缺乏明确目标感
**缓解：** 
- 设置阶段性里程碑（每10级、每次形态变化）
- 进化路线图提前展示，给予明确预期
- 成就系统提供额外目标

**风险：** 离线机制可能让玩家觉得"不需要经常打开"
**缓解：**
- 关键事件必须在线触发（形态进化、技能解锁）
- 探索日志的"开盲盒"体验鼓励频繁查看
- 每日任务提供短期目标

### 10.2 技术风险

**风险：** localStorage容量限制（通常5-10MB）
**缓解：**
- 探索日志只保留最近50条
- 数据结构精简，避免冗余
- 第二阶段迁移到云端前清理历史数据

**风险：** 浏览器兼容性（特别是iOS Safari的PWA限制）
**缓解：**
- 优先测试iOS Safari
- 避免使用实验性API
- 提供降级方案（如不支持Service Worker时仍可使用）

### 10.3 用户体验风险

**风险：** 像素风可能不符合部分用户审美
**缓解：**
- 第一阶段快速验证，根据反馈决定是否调整风格
- 确保像素风质量足够高，避免"粗糙感"

**风险：** 儿童用户可能需要更简化的界面
**缓解：**
- 移动优先设计天然简洁
- 第二阶段可考虑"儿童模式"（更大按钮、引导提示）

## 11. 风险与挑战

### 11.1 设计风险

**风险：** 无限进化可能导致缺乏明确目标感
**缓解：** 
- 设置阶段性里程碑（每10级、每次形态变化）
- 进化路线图提前展示，给予明确预期
- 成就系统提供额外目标

**风险：** 离线机制可能让玩家觉得"不需要经常打开"
**缓解：**
- 关键事件必须在线触发（形态进化、技能解锁）
- 探索日志的"开盲盒"体验鼓励频繁查看
- 每日任务提供短期目标
- AI生成的离线摘要增加回归期待感

### 11.2 技术风险

**风险：** IndexedDB/localStorage容量限制
**缓解：**
- IndexedDB优先（>50MB），localStorage降级（5-10MB）
- 探索日志只保留最近50条
- AI缓存定期清理（7天TTL）
- 数据结构精简，避免冗余
- 第二阶段迁移到云端前清理历史数据

**风险：** 浏览器兼容性（特别是iOS Safari的PWA限制）
**缓解：**
- 优先测试iOS Safari
- 避免使用实验性API
- 提供降级方案（如不支持Service Worker时仍可使用）

**风险：** Pixi.js性能问题（移动端低端设备）
**缓解：**
- 限制同屏精灵数量
- 使用精灵图合并
- 低性能设备降低帧率（30fps）
- 提供"性能模式"选项

### 11.3 AI相关风险

**风险：** AI生成内容不可控（不当内容、前后矛盾）
**缓解：**
- 输出JSON schema严格约束
- 关键词黑名单过滤
- 预设内容作为fallback
- 人工审核高频缓存内容

**风险：** API延迟影响体验（>3秒用户流失）
**缓解：**
- 非实时场景使用AI（离线日志、事件预生成）
- 设置3秒超时，超时立即降级到预设内容
- 缓存命中率目标70%+
- 显示"生成中..."加载动画

**风险：** API成本失控
**缓解：**
- 每日成本监控和告警
- 多级降级策略（Level 1→2→3）
- 缓存优先策略（90%+请求无需API）
- DAU超过阈值后迁移到本地模型

**风险：** DeepSeek API停服或政策变化
**缓解：**
- 实现多模型切换接口（DeepSeek / Gemini / 本地）
- 所有AI功能可完全降级到预设内容
- 定期测试降级流程

### 11.4 用户体验风险

**风险：** 像素风可能不符合部分用户审美
**缓解：**
- 第一阶段快速验证，根据反馈决定是否调整风格
- 确保像素风质量足够高，避免"粗糙感"

**风险：** 儿童用户可能需要更简化的界面
**缓解：**
- 移动优先设计天然简洁
- 第二阶段可考虑"儿童模式"（更大按钮、引导提示）

**风险：** DNA/性格系统过于复杂，玩家理解成本高
**缓解：**
- DNA完全隐藏，玩家只看到结果（"你的宠物天生力量型"）
- 性格通过动画和对话自然展现，不需要数值解释
- 新手引导突出"每只宠物都独一无二"的核心概念
